# Changelog

## v3.3.1 — 2026-05-25

### 改进
- 同步更新 README 描述，准确反映插件实际功能（COS 收图、文字+图片混合消息）

## v3.3.0 — 2026-05-25

### 说明
- 因 v3.2.x 系列 tag 存在冲突，版本号跳至 v3.3.0

### 改进
- 基于 v3.1.1 稳定版本重新整理发布

## v3.1.1 — 2026-05-23

### 改进
- **视频封面支持**：优化封面获取逻辑，支持从 `post.cover` 和 `vod_list` 提取视频帖子封面
- 统一 CDN 前缀补全逻辑

## v3.1.0 — 2026-05-23

### 新功能
- **图片嵌入**：简报自动附带帖子封面图，提升内容直观性
- **图片数量限制**：新增 `enable_image_limit` 和 `max_images` 配置，控制单条消息图片数量，QQ 单条合并消息更友好

### 改进
- 简报输出从纯文本改为 `MessageChain`（文本+图片混合消息）
- README 新增平台兼容性声明（主要针对 OneBot/QQ 平台优化）

## v3.0.2 — 2026-05-23

### Bugfix
- 修复配置 `categories` 存储为单条字符串时解析失败的问题
- 定时推送静默跳过时增加 warning 日志
- 启动时打印距首次推送的剩余时间
- 新增 `/米游社诊断` 命令

## v3.0.1 — 2026-05-22

### Bugfix
- 修复 `/米游社` 命令误拉取崩坏3帖子的 bug（现在只拉取配置中指定的游戏）
- 修复星穹铁道和绝区零 forum_id 互换导致的新闻错乱

## v3.0.0 — 2026-05-22

### 重大变更
- **全面转向米游社**：去掉所有联网搜索、GitHub Trending、通用分类代码
- 数据源改为 `bbs-api.miyoushe.com` 米游社 BBS API
- 支持游戏：原神、崩坏：星穹铁道、绝区零、崩坏3
- 命令改为 `/米游社`、`/米游社状态`、`/米游社管理 push`

### 移除
- 移除联网搜索（Bocha/Tavily/Brave）相关配置和代码
- 移除 AI/科技/游戏开发/热门游戏/时事热点/科技数码等通用分类
- 移除 GitHub Trending API 调用

## v2.1.0 — 2026-05-22

### 改进
- 搜索词自动拼接当天日期前缀，确保搜到当日最新新闻
- LLM prompt 强化时效性约束，优先采用日期最近的内容
- "二次元/游戏" 分类更名为 "热门游戏"，搜索词改为原神/米哈游/洛克王国/地平线6等具体方向
- LLM 输出要求每条摘要至少 2-3 句话，确保内容充实

## v2.0.0 — 2026-05-21

### 新功能
- **多分类支持**：内置 AI/科技/GitHub/游戏/二次元/时事/数码等多个分类
- **按目标个性化推送**：每个推送目标可独立指定分类，例如大号收 AI+科技，群收二次元+时事
- 搜索去重优化：多个目标共用分类时只搜一次，结果复用

### 变更
- 目标字符串格式扩展：`platform:type:id[:分类1,分类2]`
- 搜索架构重构：支持 `web`（联网搜索）和 `github`（GitHub API）两种来源类型

## v1.2.0 — 2026-05-21

### 新功能
- **自动去重**：基于 URL 缓存，已推送过的新闻不重复出现
- 增加 `dedup_ttl_days` 配置项（默认 7 天）
- 去重缓存文件 `_seen_news.json`

## v1.1.0 — 2026-05-21

### 新功能
- **联网搜索**：支持 Bocha / Tavily / Brave 搜索引擎
- 数据来源从 LLM 知识回忆切换为实时搜索

### 变更
- 搜索 API 配置：`search_provider`、`search_api_key`、`search_count`
- LLM 整理改为基于搜索结果的摘要，而非知识生成

## v1.0.0 — 2026-05-21

### 初始版本
- 基础插件框架：AstrBot Star 结构
- 定时推送 + 命令触发
- LLM 知识生成新闻简报（AI/科技/GitHub/游戏开发）
- QQ 友好 emoji 排版（非 Markdown）
